# java-movie-precourse

기능을 구현하기 전 README.md에 구현할 기능 목록을 작성하고, 구현 전략을 함께 정리한다.

1. SeatGrade : 
    - 책임  
        - [ ] 좌석 등급을 알아야한다.
        - [ ] 좌석 가격을 알아야한다.
2. Row :
    - 책임 
        - [ ] 좌석 행을 알아야한다.
    - 조건
        - 알파벳만 써야한다. ('A' ~ 'E')
3. Column :
    - 책임
        - [ ] 좌석 열을 알아야한다.
    - 조건
        - 숫자 1~4로 표시 해야한다.
4. SeatNumber :
    - 책임 
        - [ ] 좌석 번호를 알아야한다.
    - 협력
        - Row, Column
5. Seat :
    - 책임
        - [ ] 좌석 번호를 알아야한다.
        - [ ] 좌석 등급을 알아야한다.
    - 협력
        - SeatGrade, SeatNumber
6. Movie :
    - 책임
        - [ ] 영화 제목을 알아야한다.
        - [ ] 영화 러닝 타임을 알아야한다.
9. Theater :
    - 책임
        - [ ] 상영관 운영 시간을 알아야한다.
        - [ ] 상영 날짜를 알아야한다.
        - [ ] 상영 시간을 알아야한다.
        - [ ] 어떤 영화를 상영할지 알아야한다.
        - [ ] 영화의 러닝 타임을 알아야한다.
    - 협력
        - Movie
10. Reservation
    - 책임
        - [ ] 예약이 되었는지 되지 않았는지 알아야한다.
        - [ ] 영화를 알아야한다.
        - [ ] 좌석을 알아야한다.
        - [ ] 상영 날짜를 알아야한다.
        - [ ] 상영 시간을 알아야한다.
        - [ ] 중복 처리를 해야한다. // 상영 날짜와 시간이 같고 좌석이 같으면 중복 처리
    - 협력
        - Seat, Theater
11. MovieDay
    - 책임 
        - [ ] 날짜가 10, 20, 30일이면 무비데이다.
        - [ ] 무비데이면 할인이 10% 들어간다.
12. TimeBased
    - 책임
        - [ ] 11시 이전 || 19시 이후면 할인된다.
        - [ ] 할인 금액은 2000원이다.
13. UserPoint
    - 책임 
        - [ ] 포인트가 있는지 알아야한다.
        - [ ] 유저 포인트 사용 여부를 알아야한다.
14. PayType
    - 책임
        - [ ] 결제할 방법이 뭔지 알아야한다.
    - 조건
        - [ ] 카드면 5% 할인, 현금이면 2%할인이다.
15. Calculater
    - 책임
        - [ ] 무비데이인지 확인한다.
        - [ ] 조조, 심야인지 확인한다.
        - [ ] 사용자 포인트가 있는 지 없는 지 확인, 사용 여부를 확인한다.
        - [ ] 결제 방법을 알아야한다.
        - [ ] 할인 순서를 알아야한다.
    - 협력
        - MovieDay, TimeBased, UserPoint, PayType, Reservation
